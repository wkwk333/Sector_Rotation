スクリプトを作成しました。以下、配置と実行の手順です。

手順1: ファイルの配置
ダウンロードした sector_rotation_monitor.py を、普段Python作業をしているフォルダ(例: C:\Users\あなた\finance\ や ~/finance/)に置いてください。特別なフォルダ構成は不要で、output フォルダは実行時に自動作成されます。

手順2: ライブラリの確認
ターミナル(またはコマンドプロンプト)で以下を実行します。すでにyfinance環境があるなら不要です。
pip install yfinance pandas numpy

手順3: 実行
python sector_rotation_monitor.py
実行すると output/ フォルダに2つのCSVが生成されます。

sector_rotation_YYYYMMDD.csv — 日次データ本体。各ETFの調整後終値、流入側・流出側バスケット指数(初日=1.0の等ウェイト平均)、rotation_ratio(流入÷流出)、20日・60日移動平均、シグナル(1=流入側優位、-1=流出側優位)、シグナル転換フラグを全営業日分収録
sector_rotation_summary_YYYYMMDD.csv — 直近のレシオ・MA値・現在のシグナル判定・直近転換日をまとめたサマリー

どちらもBOM付きUTF-8(utf-8-sig)で出力するので、Excelでダブルクリックしても文字化けしません。

読み方
rotation_ratio が上昇していればXLF+XLI+XLEがSMH+IGVに対して優位、つまりローテーション進行中です。20日MAが60日MAを上抜けたら「流入側優位」への転換シグナル、下抜けたら「テック回帰」の転換シグナルとして機構的に判定します。

エラー対処の実装ポイント(発生源の明示)
過去に問題になりやすかった箇所は明示的に対処してあります。yfinanceの複数ティッカー取得で返るMultiIndex列は download_prices 内で判定・平坦化し、単一ティッカー時のSeries化も to_frame で吸収しています。バスケット計算(build_basket)は一部銘柄が取得失敗しても残りで計算を続行し、警告で欠損銘柄を明示します。すべてのraiseメッセージに [エラー発生源: 関数名] を付けているので、失敗時にどの段階で止まったか即座に特定できます。取得失敗時は5秒間隔で3回リトライします。

カスタマイズ
冒頭の CONFIG 辞書だけ書き換えれば、バスケット構成(例: 流出側に QQQ 追加)、取得期間、MA日数を変更できます。コード本体は触る必要はありません。
毎営業日の自動実行(タスクスケジューラ/cron登録)や、前回お見せした個別20銘柄ずつのバスケット版への拡張も可能ですので、必要でしたらお知らせください。

--------------------------------------------------
追加機能: グラフ化・複数指標監視・1枚サマリー
--------------------------------------------------

sector_rotation_monitor.py は現在、5つのペア指標(バリュー/景気循環 vs テック、
ディフェンシブ vs テック、小型 vs 大型、市場の厚み、信用リスク選好度)と
VIX指数(レベル指標)をまとめて監視する構成に拡張されています。
CONFIG["pairs"] / CONFIG["level_indicators"] に指標を追加・変更できます。

1. python sector_rotation_monitor.py
   → 各指標の日次CSVと、ペア/レベル指標それぞれのサマリーCSVを output/ に出力

2. python plot_rotation.py
   → 各ペアごとに2段組グラフ(レシオ+MA、バスケット指数)を
     output/sector_rotation_chart_<ペア名>_YYYYMMDD.png として出力

3. python generate_dashboard.py
   → 全6指標を1枚にまとめたダッシュボード画像を
     output/sector_rotation_dashboard_YYYYMMDD.png として出力
     (上段に複数指標の悪化度合いから算出した「総合判定」バナー付き)

ダッシュボードの読み方は dashboard_guide.txt を参照してください。
どの指標が何を意味するか、総合判定がどう計算されているか、
実践的な読み方の例まで説明しています。

いずれのスクリプトも実行前に sector_rotation_monitor.py の実行が必要です
(plot_rotation.py と generate_dashboard.py は output/ 内の最新CSVを読み込みます)。
matplotlib が未インストールの場合は pip install matplotlib を行ってください。

--------------------------------------------------
exeファイルでのワンクリック実行
--------------------------------------------------

上記3ステップをまとめて実行する SectorRotationDashboard.exe を
dist フォルダに用意しています。ダブルクリックするだけで
データ取得 → グラフ作成 → ダッシュボード作成 まで自動で行われ、
exe と同じ場所に output フォルダが作られます。
処理が終わると「何かキーを押すと終了します」と表示されるので、
内容を確認してから何かキーを押すとウィンドウが閉じます。

  - exe の実体   : dist\SectorRotationDashboard.exe
  - 実行結果     : (exeを置いた場所)\output\ 以下に一式出力
  - デスクトップ等に置く場合は exe ファイルだけコピーすればOKです
    (venv や .py ファイルは不要。exeに全部同梱されています)

スクリプトを修正した後にexeへ反映させたい場合は、venv環境で以下を実行して
再ビルドしてください (build_exe.ps1 が pyinstaller を呼び出します)。
    .\venv\Scripts\python.exe -m pip install pyinstaller   (初回のみ)
    .\build_exe.ps1

--------------------------------------------------
カテゴリ別代表銘柄の確認
--------------------------------------------------

python top_holdings.py

を実行すると、バリュー/景気循環・ディフェンシブ・テックの3カテゴリについて、
現在の代表銘柄(上位10銘柄、各ETFの上位保有銘柄をETF間で均等加重平均して
再ランキングしたもの)を1枚の画像にまとめて出力します。
出力先: output/06_sector_rotation_top_holdings_YYYYMMDD.png

--------------------------------------------------
出力ファイルの命名規則
--------------------------------------------------

ダッシュボード以外のPNG(ペア別グラフ・代表銘柄画像)は、先頭に2桁の番号を
付けて output/ 内で意図した順番に並ぶようにしています
(01〜05: ペア別グラフ、06: 代表銘柄画像)。ダッシュボード画像
(sector_rotation_dashboard_YYYYMMDD.png)には番号を付けていません。

--------------------------------------------------
文字化け対策について
--------------------------------------------------

console_utf8.py を各スクリプトの先頭でimportすることで、Windowsコンソールの
出力コードページをUTF-8に切り替え、PowerShellやコマンドプロンプトで
日本語が文字化けしないようにしています。venv経由でスクリプトを直接実行する
場合もexe実行の場合も、この対策は自動的に効きます。
