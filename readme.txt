米国株 資金ローテーション監視ツール
====================================

米国株のセクター間・ファクター間の資金ローテーションを、複数のETFバスケット
比率とVIX指数から定量的に追跡するツール一式です。

--------------------------------------------------
セットアップ
--------------------------------------------------

1. このフォルダに Python 仮想環境を作成します(初回のみ)。
     py -3.11 -m venv venv
2. 必要ライブラリをインストールします。
     .\venv\Scripts\python.exe -m pip install yfinance pandas numpy matplotlib

--------------------------------------------------
監視している指標
--------------------------------------------------

CONFIG["pairs"] (sector_rotation_monitor.py 冒頭) に登録されたペア指標。
いずれもレシオ = 流入側バスケット ÷ 流出側バスケット で、値が大きいほど
流入側が優位。移動平均クロス(短期MA vs 長期MA)でシグナルを判定します。

  ①value_tech      : テック(SMH+IGV) vs バリュー/景気循環(XLF+XLI+XLE)
                      → テック優位ほどレシオが増える向き
  ②defensive_tech  : テック(XLK+SMH+IGV) vs ディフェンシブ(XLP+XLU+XLV)
                      → テック優位ほどレシオが増える向き
  ③breadth_smallcap: 小型(IWM) vs 大型(SPY)
  ④breadth_equalweight: 等ウェイトS&P(RSP) vs 時価総額加重S&P(SPY)
  ⑤credit_risk     : ハイイールド社債(HYG) vs 米国債7-10y(IEF)

加えてレベル指標として ⑥VIX指数(^VIX) を平常/警戒/パニック的リスクオフの
3ゾーンで判定します(CONFIG["level_indicators"])。

①②のみ他ペアと逆向き(テック優位で増加)である点に注意してください。
③④⑤は「1つ目に書かれている側」が優位なほどレシオが増えます。

--------------------------------------------------
実行手順
--------------------------------------------------

1. python sector_rotation_monitor.py
   → 各指標の日次CSV、ペア/レベル指標それぞれのサマリーCSVを output/ に出力

2. python plot_rotation.py
   → 各ペアごとに2段組グラフ(レシオ+MA、バスケット指数)を
     output/01〜05_sector_rotation_chart_<ペア名>_YYYYMMDD.png として出力

3. python generate_dashboard.py
   → 全6指標を1枚にまとめたダッシュボード画像を
     output/sector_rotation_dashboard_YYYYMMDD.png として出力
     (総合判定バナー、転換直後・要確認バッジ、短期-長期MAスプレッド付き)

4. python top_holdings.py
   → バリュー/ディフェンシブ/テックそれぞれの代表銘柄(上位10銘柄、PERによる
     割安度の色分き)を1枚の画像として
     output/06_sector_rotation_top_holdings_YYYYMMDD.png に出力

2〜4は output/ 内の最新CSVを読み込むため、事前に 1 の実行が必要です。

まとめて実行したい場合は run_pipeline.py (1〜3を1プロセスで実行) を
使うか、後述の exe を使ってください。

--------------------------------------------------
ダッシュボードの読み方
--------------------------------------------------

dashboard_guide.txt を参照してください。各パネルの意味、色分けの意味、
総合判定の算出ロジック、実践的な読み方の例まで説明しています。

--------------------------------------------------
exeファイルでのワンクリック実行
--------------------------------------------------

sector_rotation_monitor.py → plot_rotation.py → generate_dashboard.py を
まとめて実行する SectorRotationDashboard.exe を dist フォルダに用意して
います(top_holdings.py は現状未同梱)。ダブルクリックするだけで実行でき、
exe と同じ場所に output フォルダが作られます。処理が終わると
「何かキーを押すと終了します」と表示されるので、内容を確認してから
何かキーを押すとウィンドウが閉じます。

  - exe の実体   : dist\SectorRotationDashboard.exe
  - 実行結果     : (exeを置いた場所)\output\ 以下に一式出力
  - デスクトップ等に置く場合は exe ファイルだけコピーすればOKです
    (venv や .py ファイルは不要。exeに全部同梱されています)

スクリプトを修正した後にexeへ反映させたい場合は、venv環境で以下を実行して
再ビルドしてください (build_exe.ps1 が pyinstaller を呼び出します)。
    .\venv\Scripts\python.exe -m pip install pyinstaller   (初回のみ)
    .\build_exe.ps1

--------------------------------------------------
出力ファイルの命名規則
--------------------------------------------------

ダッシュボード以外のPNG(ペア別グラフ・代表銘柄画像)は、先頭に2桁の番号を
付けて output/ 内で意図した順番に並ぶようにしています
(01〜05: ペア別グラフ、06: 代表銘柄画像)。ダッシュボード画像
(sector_rotation_dashboard_YYYYMMDD.png)には番号を付けていません。

--------------------------------------------------
カスタマイズ
--------------------------------------------------

sector_rotation_monitor.py 冒頭の CONFIG 辞書を書き換えれば、バスケット
構成・取得期間・MA日数・シグナルラベルを変更できます。コード本体は
触る必要はありません。ペアを追加・削除した場合、top_holdings.py の
CATEGORIES 定義も対応する pair 名を参照しているため、整合性を確認して
ください。

--------------------------------------------------
エラー対処の実装ポイント(発生源の明示)
--------------------------------------------------

yfinanceの複数ティッカー取得で返るMultiIndex列は download_prices 内で
判定・平坦化し、単一ティッカー時のSeries化も to_frame で吸収しています。
バスケット計算(build_basket)は一部銘柄が取得失敗しても残りで計算を
続行し、警告で欠損銘柄を明示します。すべてのraiseメッセージに
[エラー発生源: 関数名] を付けているので、失敗時にどの段階で止まったか
即座に特定できます。取得失敗時は5秒間隔で3回リトライします。

--------------------------------------------------
文字化け対策について
--------------------------------------------------

console_utf8.py を各スクリプトの先頭でimportすることで、Windowsコンソール
の出力コードページをUTF-8に切り替え、PowerShellやコマンドプロンプトで
日本語が文字化けしないようにしています。venv経由でスクリプトを直接実行
する場合もexe実行の場合も、この対策は自動的に効きます。

--------------------------------------------------
Git管理について
--------------------------------------------------

このフォルダは git init 済みです。venv/・build/・dist/・output/ は
.gitignore で除外しているため、コミット対象はソースコードと
ドキュメントのみです。
